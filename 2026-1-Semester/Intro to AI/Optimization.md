# 가천대학교 인공지능개론 Week 4: Optimization (최적화)

본 문서는 인공지능 시스템이 수많은 선택지 중에서 최적의 해를 찾아내는 **최적화(Optimization)** 기법과 **제약 조건 만족 문제(CSP)**의 핵심 원리를 다룹니다. 머신러닝의 하이퍼파라미터 튜닝이나 모델 가중치 업데이트 등 다양한 AI/보안 알고리즘의 기반이 되는 중요한 수학적, 논리적 개념들을 포함하고 있습니다.

---

## 1. 최적화(Optimization)와 지역 탐색(Local Search)

최적화란 주어진 여러 대안 중에서 가장 좋은(가장 비용이 적거나 가장 가치가 높은) 선택지를 고르는 과정입니다. 

### 지역 탐색 (Local Search)
보통 탐색 알고리즘은 시작점부터의 경로를 모두 기억하지만, 최적화 문제에서는 오직 **'목적지(Solution)'** 자체에만 관심이 있습니다. 
* **특징:** 단일 노드(현재 상태)만 유지하며, 지나온 경로를 저장하지 않습니다. 오직 현재 노드의 **주변 노드(Neighboring node)**로만 이동하며 실시간으로 탐색합니다. 메모리 효율성이 매우 뛰어납니다.
* **그림 해설 (병원 위치 선정 예시):** 교안에서는 격자 맵 위에 여러 집(파란색 집 아이콘)과 2개의 병원(빨간 십자가 아이콘)이 배치된 예시를 보여줍니다. 알고리즘은 병원의 위치를 상하좌우 주변 노드로 조금씩 이동시키면서, 모든 집에서 병원까지의 거리 합산(Cost)을 계산합니다. 그림의 진행에 따라 Cost가 17 $\rightarrow$ 15 $\rightarrow$ 13 $\rightarrow$ 11 $\rightarrow$ 9로 점점 줄어들며, 전체 거리가 최소화되는 최적의 병원 위치를 찾아냅니다.

### 상태 공간 지형 (State-Space Landscape)
현재 상태를 평가하는 기준에 따라 탐색의 목표가 달라집니다. 교안의 막대그래프 그림들은 상태(x축)에 따른 평가값(y축)을 시각화한 것입니다.
* **목적 함수 (Objective Function):** 값이 높을수록 좋습니다 (예: 모델의 정확도). 막대그래프에서 가장 높은 막대가 **전역 최댓값(Global Maximum)**이며 알고리즘의 최종 목표가 됩니다.
* **비용 함수 (Cost Function):** 값이 낮을수록 좋습니다 (예: 오차율, 비용). 막대그래프에서 가장 낮은 막대가 **전역 최솟값(Global Minimum)**이 됩니다.
* 현재 상태(Current State)에서 주변에 갈 수 있는 노드(Neighbors)를 탐색하며 그래프를 타고 오르거나 내려갑니다.

*(참고: 교안 1p ~ 29p)*

## 2. 언덕 오르기 알고리즘 (Hill Climbing)

현재 상태에서 인접한 이웃 노드들을 평가한 후, **가장 가치가 높은(최고의) 이웃**으로만 이동하는 탐색 방법입니다.

### 동작 원리 및 한계점
1. 현재 노드의 이웃 중 가장 높은 값을 가진 노드를 선택합니다.
2. 선택한 이웃의 값이 현재 노드보다 높지 않다면 탐색을 종료하고 현재 값을 반환합니다.
3. 이웃이 더 높다면 현재 노드를 이웃 노드로 갱신하고 반복합니다.

하지만 이 방식은 치명적인 한계가 있습니다. 순간적인 선택으로는 Best를 향해 가지만, 궁극적으로 그래프의 전체를 보지 못하기 때문입니다.
* **지역 최댓값 (Local Maxima):** 주변보다는 높지만 전체 그래프 기준으로는 최고점이 아닌 봉우리에 갇히는 현상입니다. 막대그래프에서 한 봉우리의 꼭대기(파란색 막대)에 도달하면, 양옆의 이웃이 모두 자신보다 낮기 때문에 전역 최댓값(초록색 막대)이 다른 곳에 있음에도 탐색을 멈춰버립니다.
* **평탄한 지역 최댓값 (Flat Local Maximum) & 어깨(Shoulder):** 값이 평평하게 이어지는 구간(여러 변수가 동일한 값을 가짐)에 도달하면 더 이상 상승할 수 있는 방향을 찾지 못해 알고리즘이 방향성을 잃거나 조기 종료될 수 있습니다.

### 언덕 오르기의 변형 (Hill Climbing Variants)
| 변형 알고리즘 | 설명 |
| :--- | :--- |
| **Steepest-ascent** | 주변 이웃 중 **가장 높은 값**을 선택하여 이동 (기본 형태). |
| **Stochastic** | 더 높은 값을 가진 이웃들 중에서 **확률적으로 무작위** 선택. (다양성 확보) |
| **First-choice** | 무작위로 이웃을 확인하다가, 현재보다 높은 값이 발견되면 **처음으로 찾은 것**을 바로 선택 (빠른 탐색). |
| **Random-restart** | 무작위 초기 상태에서 언덕 오르기를 여러 번 반복하여 Local Maxima 문제를 회피. |
| **Local beam search** | 단일 노드가 아닌 **k개의 노드**를 동시에 유지하며, 가장 높은 가치의 이웃들을 탐색해 다양성을 극대화. |

*(참고: 교안 30p ~ 50p)*

## 3. 담금질 기법 (Simulated Annealing)

Hill Climbing이 Local Maxima에 빠지는 문제를 해결하기 위해 고안된 알고리즘입니다. 금속공학의 담금질(고온에서 서서히 냉각시켜 구조를 단단하게 만드는 과정)에서 착안했습니다.

### 핵심 개념: "현재보다 안 좋은 상태로의 이동 허용"
알고리즘 초기에는 일정 확률로 값이 낮아지는(나빠지는) 방향으로의 이동을 허용하여 좁은 봉우리를 탈출할 기회를 줍니다. 
* **온도(Temperature, $T$):** 탐색 초반에는 온도($T$)가 높게 설정됩니다. 이 시기에는 현재보다 가치가 떨어지는 이웃도 높은 확률로 채택합니다. 시간이 지날수록 온도는 점차 낮아지며($T \rightarrow 0$), 후반부에는 점차 Hill Climbing처럼 더 좋은 상태로만 이동하게 됩니다.
* **확률 수식:** 이동할 이웃과의 가치 차이를 $\Delta E$라고 할 때, $\Delta E > 0$(개선됨)이면 100% 이동합니다. 만약 $\Delta E \le 0$(나빠짐)이더라도 **$e^{\Delta E/T}$ 의 확률**로 이동을 수락합니다.

**그림 해설:** 교안의 그래프(MakeAGIF.com 출처)는 굴곡이 심한 함수 공간(Feasible Solutions)에서 파란색 점이 이리저리 튀어 오르는 모습을 보여줍니다. 이는 온도($T$)를 통해 무작위성을 부여받아 Local Minima(작은 골짜기)를 '격렬하게 흔들어서' 빠져나오고, 최종적으로 가장 깊은 Global Minima(최적해)로 찾아 들어가는 과정을 시각적으로 나타낸 것입니다. 

*(참고: 교안 51p ~ 71p)*

## 4. 선형 계획법 (Linear Programming)

순수 수학적 함수와 제약조건을 수식으로 모델링하여 해를 찾는 최적화 방식입니다. 

### 구성 요소 및 공식화 예시
* **목적 함수 (Cost/Objective Function):** 최적화할 대상. 
    * *예시:* 공장 기계 $X_1$은 시간당 50달러, $X_2$는 시간당 80달러가 든다면 비용 최소화 함수는 $50x_1 + 80x_2$ 가 됩니다.
* **제약 조건 (Constraints):** $a_1x_1 + a_2x_2 + ... \le b$ 또는 $= b$ 형태의 선형 방정식.
    * *예시:* 사용 가능한 노동력이 총 20단위이고 $X_1$이 5단위, $X_2$가 2단위를 쓴다면, 제약식은 $5x_1 + 2x_2 \le 20$ 입니다.
    * 목표 생산량이 90단위 이상이어야 한다면 $10x_1 + 12x_2 \ge 90$이 되며, 이를 통일된 폼($\le$)으로 맞추기 위해 음수를 곱해 $-10x_1 - 12x_2 \le -90$ 으로 변환하여 공식화합니다.
* **해결 알고리즘:** Simplex 알고리즘, Interior-Point 알고리즘 등을 통해 다면체 그래프의 꼭짓점 공간을 순회하며 최적해를 도출합니다.

*(참고: 교안 72p ~ 77p)*

## 5. 제약 조건 만족 문제 (Constraint Satisfaction Problems, CSP)

알고리즘의 논리를 상태(State) 기반이 아닌, 변수와 제약 조건 기반으로 정의하여 해결하는 방식입니다. 모든 변수가 주어진 제약 조건을 위배하지 않고 특정 값을 할당받으면 해결됩니다.

### CSP의 3대 요소
1.  **Variables (변수):** 값을 할당받아야 하는 주체. 예) $\{X_1, X_2, ..., X_n\}$
2.  **Domains (도메인):** 각 변수가 가질 수 있는 값의 후보군. 예) $\{D_1, D_2, ..., D_n\}$
3.  **Constraints (제약 조건):** 변수들 간의 관계 규칙 집합. (Hard Constraint: 반드시 만족해야 함 / Soft Constraint: 선호도의 문제)

### 교안 예시 (시험 시간표 짜기)
* **상황:** 4명의 학생(1~4)이 각각 3과목씩 수강 중이며, 시험 시간 슬롯은 월(Mon), 화(Tue), 수(Wed) 3개가 있습니다. 한 학생이 같은 날짜에 2개의 시험을 칠 수 없습니다.
* **Variables:** 시험을 치르는 총 7개의 과목 노드 $\{A, B, C, D, E, F, G\}$
* **Domains:** $\{Mon, Tue, Wed\}$
* **Constraints:** 학생 1은 A, B, C를 수강하므로 이 과목들은 서로 다른 날이어야 합니다. 즉, $A \ne B, A \ne C, B \ne C$. 이런 식으로 그래프를 그리면, 겹치는 학생이 있는 과목끼리는 간선(Edge)으로 연결된 구조가 됩니다.

### 일관성 (Consistency)
* **단항 제약 (Unary Constraint) & Node Consistency:** 단일 변수에 대한 제약. (예: 과목 A는 월요일에 볼 수 없다 $\rightarrow \{A \ne Mon\}$). 변수의 도메인 내 모든 값이 해당 변수의 단항 제약을 만족하면 '노드 일관성'을 갖춘 것입니다. 과목 A의 도메인 $\{Mon, Tue, Wed\}$에서 $Mon$을 삭제하여 $\{Tue, Wed\}$만 남깁니다.
* **이항 제약 (Binary Constraint):** 두 변수 간의 제약. (예: $A \ne B$).

*(참고: 교안 78p ~ 104p)*

## 6. 호 일관성 (Arc Consistency)과 역추적 탐색 (Backtracking Search)

### 호 일관성 (Arc Consistency) 및 AC-3 알고리즘
변수 X가 변수 Y에 대해 '호 일관성'을 가진다는 것은, **X의 도메인에 있는 모든 값에 대해, 제약조건을 만족하는 짝(pair)이 Y의 도메인에 최소 하나 이상 존재한다**는 뜻입니다.
* *예시:* 도메인이 $A = \{Tue, Wed\}$, $B = \{Wed\}$이고 제약조건이 $A \ne B$일 때, A가 Wed를 선택하면 B에서 선택할 수 있는 값이 없습니다. 따라서 A의 도메인에서 Wed를 삭제하여 $A = \{Tue\}$ 로 축소해야 Arc Consistency가 성립합니다.
* **AC-3 알고리즘:** Queue 구조를 사용하여 CSP 내의 모든 호(간선)를 순회하며 제약조건을 만족하지 못하는 도메인 값을 삭제합니다. 만약 특정 변수의 도메인 크기가 0이 되면 해가 없는 것(False)으로 판별합니다.

### 역추적 탐색 (Backtracking Search)
CSP를 트리 탐색 문제로 모델링하여 푸는 기본 알고리즘입니다. 변수에 값을 하나씩 할당(Assignment)해 보고, 제약 조건을 위배하면 이전 상태로 되돌아가(Backtrack) 다른 값을 시도합니다.
* **탐색 시각화 해설:** 교안의 그림에서는 A 과목에 Mon을 할당하고, B에 Mon을 시도하다 제약 위배로 실패, Tue로 변경하여 진행하는 깊이 우선 탐색(DFS) 트리의 동작 과정을 순차적으로 보여줍니다. 막히면 노드를 버리고 부모 노드로 돌아가 다음 후보 도메인(Wed 등)을 시도합니다.

### 휴리스틱 최적화 기법 (Heuristics)
Backtracking의 탐색 속도를 극적으로 높이기 위해 어떤 변수와 값을 먼저 선택할지 결정하는 규칙입니다.
1.  **변수 선택 (SELECT-UNASSIGNED-VAR)**
    * **MRV (Minimum Remaining Values):** 도메인에 **남은 값의 수가 가장 적은 변수**를 먼저 선택합니다. (실패를 빨리 발견하여 가지치기 가능)
    * **Degree Heuristic:** 다른 변수들과 **가장 많은 제약조건(간선)을 가진 변수**를 먼저 선택합니다.
2.  **값 선택 (DOMAIN-VALUES)**
    * **Least-Constraining Values:** 인접한 이웃 변수들의 도메인 선택지를 **가장 적게 제한하는 값**을 우선적으로 할당합니다.
3.  **추론 (Inference: MAC)**
    * 새로운 값을 할당할 때마다 AC-3 알고리즘을 호출하여 주변 변수들의 호 일관성(Arc-consistency)을 즉시 갱신합니다(MAC; Maintaining Arc-Consistency). 이를 통해 미리 불가능한 경로를 사전에 차단합니다.

*(참고: 교안 105p ~ 185p)*
