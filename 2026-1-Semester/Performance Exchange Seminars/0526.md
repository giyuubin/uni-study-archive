# 노용환
## LLM 기초
- LLM이 하는 일은 단 하나: 다음 토큰 예측
- LLM은 확신하며 틀린다.
  - 검증은 사람의 몫
- 작업 시간이 30분을 넘으면 터미널 에이전트를 써라.
- 걍 Claude code 써라.
- 프롬프트 엔지니어링: 잘 물어봐야, 잘 대답한다.
  - LLM은 내 의도를 추측할 뿐이다.
  - 프롬프트의 4가지 구성요소:
    - Role: 어떤 관점에서 답해야 하는가?
    - Context: 알아야 할 배경 정보
    - Task: 무엇을 해야 하는가?
    - Output Format: 결과물의 형식
  - 명확하게 단계별로 정확히 집어줘야함.
- 토큰관리:
  - 컨텍스트가 꽉 차면 LLM은 잊기 시작한다.
  - 세션을 작업 단위로 나눠라
  - 핵심 3가지:
    - 컨텍스트 오염,..
- Copilot & Cursor
  - 프롬프트 안에서 전부 해결하려면 안됨.
  - 핵심 4가지:
    - 속도의 함정: 인간이 전부 검증해야됨
    - 컨텍스트 범위
    - 설정 파일 3종
    - 도구 분담
  - 걍 터미널 기반을 써라.
