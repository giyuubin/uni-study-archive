# 인공지능 프로그래밍: 군집화(Clustering) 핵심 개념 및 알고리즘

## 1. 군집화(Clustering)의 개요

### 1.1. 비지도 학습과 군집화
머신러닝은 크게 정답(Label)의 유무에 따라 지도 학습과 비지도 학습으로 나뉩니다. **군집화(Clustering)**는 정답이 없는 상태에서 데이터의 특성과 패턴만을 이용해 데이터를 여러 그룹으로 나누는 대표적인 **비지도 학습(Unsupervised Learning)**입니다.

- **목적**: 주어진 데이터들을 유사한 특징을 가진 그룹(Cluster)으로 묶어 데이터의 숨겨진 구조를 파악하는 것입니다.
- **핵심 원리**: 
  - **응집도(Intra-cluster distance) 최소화**: 같은 군집 내에 속한 데이터들끼리의 거리는 최대한 가깝게(비슷하게) 만듭니다.
  - **분리도(Inter-cluster distance) 최대화**: 서로 다른 군집에 속한 데이터들 간의 거리는 최대한 멀게(다르게) 만듭니다.

### 1.2. 분류(Classification) vs 군집화(Clustering)
- **분류 (지도 학습)**: 사전에 클래스의 개수와 레이블(정답)을 알고 있습니다. 새로운 입력값이 들어왔을 때, 이를 어떤 정답 클래스로 매핑할지 예측하는 함수(Decision Boundary)를 찾는 것이 목표입니다. (예: 어른과 아이의 데이터를 분류하는 기준선 찾기)
- **군집화 (비지도 학습)**: 군집의 개수나 각 데이터가 어디에 속하는지 모릅니다. 데이터 포인트들의 분포(예: 키와 몸무게)를 보고, 자연스럽게 모여있는 '어른 특성 그룹'과 '아이 특성 그룹'으로 구조화하는 것이 목표입니다.

*(참고: 교안 1p ~ 5p)*

## 2. 군집화의 활용 및 수행 과정

### 2.1. 군집화의 주요 활용 분야
방대한 양의 데이터를 요약(Summarization)하고 시각화(Visualization)하는 데 매우 유용합니다.
1. **문서 군집화 (Document Clustering)**: 유사한 주제나 텍스트를 가진 문서들을 그룹화합니다.
2. **유전자 군집화 (Gene Clustering)**: 비슷한 기능을 하는 유전자들을 묶습니다.
3. **주식/금융 (Stocks)**: 가격 변동성이나 6개월 수익률 등을 바탕으로 주식 포트폴리오를 구성하고 관리합니다. (수익률 극대화 및 변동성 최소화 전략에 활용)
4. **고객 마케팅 (Customer Marketing)**: 고객의 구매 패턴을 분석하여 맞춤형 타겟 마케팅을 수행합니다.
5. **지표 기반 시각화**: 예컨대 빈곤 관련 지표를 바탕으로 국가들을 자기조직화지도(SOM, 육각형 그리드 형태) 상에 배치하여, 유사한 경제 수준의 국가들이 인접하도록 시각화할 수 있습니다.

### 2.2. 군집화 표준 수행 절차 (Standard Clustering Procedure)
성공적인 군집화를 위해서는 다음의 단계적 파이프라인을 거칩니다.
1. **특성 선택 (Feature Selection)**: 전체 데이터 베이스에서 군집화에 사용할 의미 있는 데이터 피처를 추출합니다.
2. **알고리즘 선택 (Algorithm Selection)**: 데이터의 형태에 맞는 적절한 군집화 알고리즘(K-Means, DBSCAN 등)을 선택하여 연산을 수행합니다.
3. **결과 검증 (Validation of results)**: 형성된 군집이 내부 응집도와 외부 분리도를 잘 충족하는지 평가 지표를 통해 타당성을 검증합니다.
4. **결과 해석 및 지식 도출 (Interpretation -> Knowledge)**: 최종 군집을 해석하여 실질적인 비즈니스 인사이트나 도메인 지식을 얻습니다.

*(참고: 교안 6p ~ 10p)*

## 3. 군집화 평가 지표 (Clustering Validity Measures)

비지도 학습의 특성상 "절대적으로 완벽한" 글로벌 정답은 존재하지 않기 때문에, 군집화가 잘 되었는지 평가하는 지표가 필요합니다. 평가는 크게 정답이 있을 때 비교하는 **외부 평가(External)**와 군집 내부의 밀집도를 보는 **내부 평가(Internal)**, 그리고 밀집도와 군집 간 거리를 동시에 고려하는 **상대적 평가(Relative)**로 나뉩니다. 대표적인 상대적 평가 지표는 다음과 같습니다.

### 3.1. Dunn Index
군집 간의 거리는 최대한 멀리, 군집 내 데이터 간의 거리는 최대한 가깝게 만드는 것을 목표로 하는 지표입니다.
- **수식**: $I(\mathcal{C}) = \frac{\min_{i \neq j} \{ d_C(C_i, C_j) \}}{\max_{1 \le l \le k} \{ \Delta(C_l) \}}$
- **분자 (Inter-cluster)**: $d_C(C_i, C_j)$는 서로 다른 두 군집 $C_i$와 $C_j$ 사이의 최소 거리를 의미합니다. 이 값이 커질수록 군집들이 잘 분리된 것입니다.
- **분모 (Intra-cluster)**: $\Delta(C_l)$는 동일한 군집 $C_l$ 내에서 가장 멀리 떨어져 있는 두 데이터 간의 거리(군집의 지름)를 의미합니다. 이 값이 작을수록 군집이 오밀조밀하게 모여있는 것입니다.
- **해석**: 분자는 커지고 분모는 작아질수록 좋으므로, **Dunn Index 값은 클수록($I(\mathcal{C}) \rightarrow \max$) 좋은 군집화 결과**를 의미합니다.

### 3.2. 실루엣 점수 (Silhouette Score)
개별 데이터 포인트 $i$가 자신이 속한 군집에 얼마나 잘 맞고, 다른 군집과는 얼마나 떨어져 있는지를 수치화한 지표입니다. (-1 ~ 1 사이의 값을 가짐)
- **$a(i)$**: 데이터 $i$와 **같은 군집** 내에 있는 다른 모든 데이터들과의 평균 거리 (군집 내 응집도, 작을수록 좋음).
- **$b(i)$**: 데이터 $i$와 **가장 가까운 인접 군집** 내의 모든 데이터들과의 평균 거리 (군집 간 분리도, 클수록 좋음).
- **수식**: $s(i) = \frac{b(i) - a(i)}{\max\{a(i), b(i)\}}$

**[계산 예시 해설]**
어떤 포인트 $i$가 속한 군집에 4개의 다른 점이 있고 이들과의 거리가 각각 0.94, 0.86, 1.29, 1.02라고 가정해 봅시다.
1. **$a(i)$ 계산**: $(0.94 + 0.86 + 1.29 + 1.02) / 4 = 1.03$
2. **$b(i)$ 계산**: $i$ 주변에 두 개의 다른 군집이 있다고 가정합니다. 첫 번째 인접 군집(점 6개)과의 평균 거리는 11.57이 나왔고, 두 번째 인접 군집(점 4개)과의 평균 거리는 5.60이 나왔습니다. 이 중 최솟값을 취하므로 $b(i) = 5.60$이 됩니다.
3. **$s(i)$ 산출**: $(5.60 - 1.03) / \max(1.03, 5.60) = 4.57 / 5.60 = 0.82$. 실루엣 점수 0.82는 1에 가까우므로 매우 군집화가 잘 되었다고 평가할 수 있습니다.

*(참고: 교안 11p ~ 17p)*

## 4. 군집화 알고리즘의 분류

군집에 속하는 방식을 기준으로 데이터가 단 하나의 군집에만 엄격하게 속하는 **Hard Clustering (Non-overlapping)**과, 확률적으로 여러 군집에 겹쳐서 속할 수 있는 **Soft Clustering (Overlapping)**으로 나뉩니다.
대표적인 Hard Clustering 알고리즘 카테고리는 다음과 같습니다.

1. **분할 기반 (Partitioning-Based)**: 사전에 설정한 K개의 중심으로 데이터를 분할합니다. (예: K-means, K-medoids, PAM)
2. **계층 기반 (Hierarchical-Based)**: 거리나 유사도를 기반으로 계층적 트리 구조를 만듭니다. (예: BIRCH, CURE)
3. **밀도 기반 (Density-Based)**: 데이터가 밀집된 공간을 군집으로 판별하며, 노이즈 제어에 탁월합니다. (예: DBSCAN, OPTICS)
4. **그리드 기반 (Grid-Based) / 모델 기반 (Model-Based)**: 공간을 격자로 나누거나 통계적 모델(EM 등)을 사용합니다.

*(참고: 교안 18p)*

## 5. K-Means 클러스터링

### 5.1. K-Means의 개념과 목적 함수
K-Means 알고리즘은 분할 기반 군집화의 대표 격으로, **사용자가 사전에 군집의 개수(K)를 지정(K must be specified)**해야 합니다. 
각 군집은 중심점(Centroid)을 가지며, 모든 데이터 포인트는 가장 가까운 중심점의 군집에 할당됩니다.
- **목적 함수**: 각 군집의 중심점 $c_i$와 해당 군집에 속한 데이터 포인트 $x_j$ 사이의 거리의 제곱합을 최소화하는 방향으로 학습합니다.
  $\arg \min_C \sum_{i=1}^{K} \sum_{x_j \in C_i} ||x_j - c_i||^2$

### 5.2. 알고리즘 수행 절차 (KMC Procedure)
1. **초기화**: 데이터 공간에 무작위(Random)로 K개의 초기 중심점(Centroid)을 설정합니다.
2. **할당 (Assignment)**: 모든 데이터 포인트를 자신과 가장 가까운 중심점을 찾아 해당 군집으로 배정합니다.
3. **업데이트 (Update)**: 새롭게 형성된 군집 내 데이터들의 평균 좌표를 계산하여, 중심점(Centroid)의 위치를 이동시킵니다.
4. **반복 및 종료**: 위 2번과 3번 과정을 반복하며, **모든 중심점의 위치가 변하지 않고 데이터의 군집 소속이 변하지 않을 때(수렴)** 알고리즘을 종료합니다.

### 5.3. 초기 중심점의 영향 (Effect of initial centroid)
중심점을 무작위로 선택하기 때문에, 초기화 위치에 따라 최종 군집화 결과가 달라질 수 있는 **지역 최솟값(Local Minimum)** 문제에 빠질 위험이 있습니다. 반복(Iteration)이 진행됨에 따라 점차 중심점이 무리의 중앙으로 이동하며 응집도(Compactness)가 상승하지만, 이상적인 전역 최적해를 보장하지는 않습니다.

### 5.4. K-Means 알고리즘의 한계점 (Limitations)
데이터의 형태에 따라 K-Means가 제대로 군집화를 수행하지 못하는 대표적인 케이스들이 도식화되어 있습니다.
1. **군집의 크기(Size)가 서로 다를 때**: 큰 크기의 성긴 데이터 그룹과 작고 조밀한 데이터 그룹이 섞여 있을 때, 단순히 거리 기반으로 잘라버려 하나의 큰 군집이 부자연스럽게 분할되는 현상이 발생합니다.
2. **군집의 밀도(Density)가 서로 다를 때**: 고밀도 군집과 저밀도 군집이 인접해 있으면, 중심점 간의 거리 분배 문제로 인해 밀도가 높은 군집의 일부 데이터가 밀도가 낮은 다른 군집에 잘못 병합되기도 합니다.
3. **구형(Spherical)이 아닌 기하학적 형태일 때**: 데이터가 U자형이나 두 개의 초승달(Moons)처럼 비선형적으로 분포하는 경우, 선형적인 거리 개념을 쓰는 K-Means는 반달 모양을 가로질러 쪼개버려 전혀 의미 없는 군집을 생성합니다.

*(참고: 교안 19p ~ 45p)*

## 6. 계층적 클러스터링 (Hierarchical Clustering)

### 6.1. 개념 및 특징
계층적 군집화는 K-Means와 달리 **사전에 군집 개수 K를 지정할 필요가 없습니다.** 데이터들 간의 유사도를 바탕으로 점진적으로 병합하거나 분할하면서 **덴드로그램(Dendrogram, 계층적 트리 구조)**을 형성합니다. 완성된 덴드로그램을 적절한 높이에서 자름(Cutting)으로써 원하는 개수의 군집을 얻어냅니다.
- **상향식 (Agglomerative)**: 각각의 데이터를 개별적인 하나의 군집으로 간주하고 시작하여, 가장 가까운 군집끼리 차근차근 병합(Merge)해 올라가 최종적으로 하나의 거대한 군집을 만드는 방식입니다. (가장 널리 쓰임)
- **하향식 (Divisive)**: 반대로 모든 데이터가 포함된 하나의 큰 군집에서 시작해, 멀리 있는 데이터를 분할해 내려가는 방식입니다.

### 6.2. 군집 간 거리 측정 방식 (Linkage Methods)
상향식 클러스터링에서 두 군집 사이의 거리를 구하는 방식에 따라 군집화 양상이 달라집니다.
1. **Single Linkage (최단 연결법)**: 두 군집에 속한 데이터들 중 **가장 가까운(Minimum Distance)** 두 데이터 사이의 거리를 군집 간 거리로 정의합니다.
2. **Complete Linkage (최장 연결법)**: 두 군집에 속한 데이터들 중 **가장 먼(Maximum Distance)** 두 데이터 사이의 거리를 군집 간 거리로 정의합니다.
3. **Average Linkage (평균 연결법)**: 두 군집에 속한 모든 데이터 쌍들의 거리의 평균(Average Distance)을 사용합니다.
4. **Centroid Linkage (중심 연결법)**: 두 군집의 기하학적 중심점(Centroid) 간의 거리를 사용합니다.
5. **Ward Method (와드 연결법)**: 두 군집을 병합했을 때, 병합 전후의 **오차 제곱합(SSE, Squared Error) 증가량이 가장 적은 군집끼리 병합**하는 방법입니다.
   - *예시 계산*: 병합 전 8개 개별 점들의 중심 오차 제곱합이 8이었습니다. 4개씩 두 그룹으로 묶은 뒤 중앙값과의 분산을 구해보니 오차 제곱합이 45가 되었습니다. 이때 Ward Distance는 $45 - 8 = 37$이 됩니다. 이 차이값이 작을수록 응집도가 높다고 판단하여 우선적으로 병합합니다.

### 6.3. 상향식 계층 군집화 수행 절차 및 예시
**[데이터 준비 및 거리 행렬 계산]**
6개의 2차원 데이터 포인트(P1~P6)가 주어졌을 때, 유클리디안 거리 공식($\sqrt{(x-a)^2 + (y-b)^2}$)을 사용해 모든 점들 간의 `N x N 거리 행렬(Distance Matrix)`을 생성합니다.
- 예: P1(0.40, 0.53)과 P2(0.22, 0.38)의 거리는 $\sqrt{(0.18)^2 + (0.15)^2} = 0.23$ 입니다.

**[병합 알고리즘 반복]**
1. **최소 거리 탐색**: 거리 행렬에서 대각선(0)을 제외하고 가장 값이 작은 데이터 쌍을 찾습니다. (예: P3와 P6의 거리가 0.11로 최소)
2. **병합**: 가장 가까운 두 포인트를 하나의 군집 (P3, P6)으로 묶습니다.
3. **행렬 갱신(Update)**: Single Linkage(최단 연결법)를 사용한다면, 새로운 군집 (P3, P6)과 다른 포인트(예: P1) 사이의 거리는 기존 $d(P3, P1)=0.22$와 $d(P6, P1)=0.23$ 중 더 작은 값인 0.22로 갱신됩니다.
4. 이를 바탕으로 갱신된 거리 행렬에서 다시 최솟값(P2와 P5 거리 0.14)을 찾아 (P2, P5)를 병합하는 식의 과정을 전체 데이터가 하나로 묶일 때까지 반복합니다.
5. **덴드로그램 작성**: 이 병합 기록을 시각화하여, X축에는 데이터를 배치하고 Y축의 병합 거리(Distance) 지점에 맞추어 연결선을 그리면 덴드로그램이 완성됩니다.

*(참고: 교안 46p ~ 79p)*

## 7. 밀도 기반 클러스터링 (DBSCAN)

### 7.1. 개념 및 필요성
기존 K-Means나 계층적 클러스터링의 한계점(비구형 데이터 처리 불가, 노이즈 취약)을 극복하기 위해 등장한 알고리즘입니다. **DBSCAN (Density-Based Spatial Clustering of Applications with Noise)**은 **"데이터가 밀집된 곳이 곧 군집이다"**라는 아이디어에 착안하여 임의의 형태(Arbitrarily shaped)를 가진 군집을 잘 찾아내고, 밀도가 낮은 곳의 데이터는 **노이즈(Noise)**로 판단하여 걸러냅니다.

### 7.2. 핵심 파라미터 및 데이터 포인트의 종류
- **Eps ($\epsilon$)**: 기준점이 되는 특정 데이터로부터의 반경(거리).
- **MinPts**: 하나의 군집을 형성하기 위해 $\epsilon$ 반경 내에 들어와야 하는 최소 데이터 개수.

이 두 파라미터를 기준으로 데이터를 3가지 유형으로 분류합니다. (가정: MinPts = 4)
1. **Core Point (핵심점)**: 자신의 Eps 반경 내에 자신을 포함하여 MinPts 이상의 데이터(4개 이상)를 가진 점입니다. 즉, "밀도가 높은" 곳에 위치한 핵심점입니다.
2. **Border Point (경계점)**: 자신의 Eps 반경 내의 데이터가 MinPts보다 적지만, 다른 Core Point의 Eps 반경 내에는 포함되는 점입니다. 군집의 외곽 테두리를 형성합니다.
3. **Noise Point (노이즈점)**: Core Point도 아니고 Border Point도 아닌 점. 즉, 반경 내에 데이터도 부족하고 다른 Core Point와도 연결되지 않은 이상치입니다.

### 7.3. 도달 가능성과 연결성 (Reachability & Connectivity)
DBSCAN은 군집을 확장해 나가기 위해 다음의 관계 정의를 사용합니다.
1. **Directly density-reachable (직접 도달 가능)**: 점 $p$가 점 $q$의 반경 반경 내에 존재하고, 점 $q$가 Core Point 조건을 만족할 때 $p$는 $q$로부터 '직접 도달 가능'하다고 합니다. (주의: 방향성이 있어 대칭적이지 않습니다. Core에서 Border로는 도달 가능하지만 반대는 성립하지 않음).
2. **Density-reachable (밀도 도달 가능)**: $q$에서 시작해 직접 도달 가능한 점들이 체인(Chain) 형태로 꼬리를 물고 이어져 결국 $p$에 닿을 수 있는 관계입니다.
3. **Density-connected (밀도 연결)**: 임의의 점 $v$로부터 $p$와 $q$가 둘 다 Density-reachable 하다면, $p$와 $q$는 밀도적으로 연결되어 있다고 봅니다.

최종적으로 DBSCAN에서 정의하는 **군집(Cluster)**은, 어떤 점에서 밀도 도달 가능한 모든 점들을 포함하여 최대화(Maximality)되며, 군집 내의 모든 점들은 서로 밀도 연결(Connectivity)되어 있어야 합니다.

### 7.4. 알고리즘 수행 절차
1. 데이터베이스에서 임의의 방문하지 않은 포인트 $p$를 선택합니다.
2. $p$의 반경 내에 MinPts 이상의 점이 있는지 확인합니다.
   - 충족한다면 $p$를 Core Point로 지정하고 새로운 군집 생성을 시작하며, 반경 내의 점들을 탐색 큐에 넣고 계속해서 도달 가능한 점들을 찾아 군집을 확장합니다.
   - 충족하지 못하면 일단 Noise로 표시하고 다음 점으로 넘어갑니다. (나중에 다른 Core Point의 경계점으로 판명되면 소속이 바뀔 수 있습니다.)
3. 모든 데이터 포인트를 방문할 때까지 이 과정을 반복합니다.

### 7.5. 주요 군집화 알고리즘 비교 (요약)
- **K-Means**: 가장 빠르고 직관적이나, 구형이 아닌 형태와 노이즈에 매우 취약하며 사전에 K를 알아야 하는 한계가 있습니다.
- **Agglomerative**: 덴드로그램을 통해 유연하게 군집 수를 결정할 수 있고 다양한 거리 척도 활용이 가능하나, 데이터가 많아질수록 연산 비용이 매우 비싸며 아웃라이어 처리가 어렵습니다.
- **DBSCAN**: 이상치(Noise) 제거에 탁월하고 데이터 분포 모양에 구애받지 않으며 K를 지정할 필요가 없습니다. 하지만 Eps와 MinPts 두 파라미터 값 설정에 매우 민감하며, **데이터 그룹 간 밀도 차이가 클 경우** 한 번의 설정으로 모든 군집을 완벽히 찾아내기 어렵다는 단점이 있습니다.

*(참고: 교안 80p ~ 95p)*
