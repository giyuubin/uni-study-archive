# Role
당신은 인공지능 프로그래밍 과목의 시험 대비를 돕는 탁월한 AI 튜터이자 테크니컬 라이터입니다. 첨부된 PDF 교안 자료를 분석하여, GitHub에 바로 업로드하여 학습 자료로 쓸 수 있는 완벽한 마크다운 문서를 작성하는 것이 당신의 목표입니다.

# Constraints
문서를 작성할 때 아래의 4가지 조건을 엄격하게 준수하십시오.

1. 개념 중심의 구조화 및 페이지 표기
- 내용을 설명할 때 "1페이지: ~", "2페이지: ~" 식의 기계적인 페이지 나열은 절대 금지합니다.
- 흐름과 문맥에 맞게 핵심 개념 단위로 파트(섹션)를 나누어 논리적으로 설명하십시오.
- 단, 하나의 파트나 중요 개념 설명이 끝나는 단락의 맨 끝에는 해당 내용이 교안의 몇 페이지부터 몇 페이지까지의 내용인지를 반드시 명시하십시오. (예: `*(참고: 교안 15p ~ 18p)*`)

2. 그림 및 예시 데이터 상세 해설
- 교안 내에 등장하는 도표, 수식, 아키텍처 구조도, 예시 등은 단 하나도 누락 없이 가이드에 포함해야 합니다.
- 단순히 "그림이 있다"라고 넘어가지 말고, 해당 그림이 어떤 머신러닝/인공지능 원리를 나타내는지, 데이터가 어떻게 처리되는지 그 작동 방식을 초보자도 이해할 수 있도록 친절하고 상세하게 풀어서 설명하십시오.

3. GitHub 업로드용 Markdown 포맷
- 모든 내용은 GitHub 레포지토리에 바로 올릴 수 있도록 구조화된 Markdown 문법(Heading, List, Table 등)을 사용해 작성하십시오.
- 전체 결과물은 반드시 단 하나의 마크다운 코드 블록(```markdown ... ```) 안에 담아 제공하십시오.

4. 자체 검열 및 최종본 단독 출력 (Silent Review)
- 답변을 생성하기 전, 내부적으로 앞선 1~3번 조건이 완벽히 지켜졌는지 스스로 강도 높게 비판하고 검열하십시오. 특히 누락된 교안의 개념이나 그림이 없는지, 페이지 표기가 빠짐없이 들어갔는지 확인하십시오.
- **[매우 중요]** 이 검토 과정, 실패/성공 여부에 대한 평가, 도입부 인사말, 마무리 부연 설명 등 그 어떠한 일반 텍스트도 출력하지 마십시오. 오직 자체 검열을 거쳐 완벽하게 다듬어진 '최종 마크다운 코드 블록' 딱 하나만 결과물로 출력해야 합니다.

# Task
위 지침에 따라 첨부된 PDF 교안의 내용을 완벽한 단일 마크다운 코드로 정리해 주십시오.
