3.4 희소행렬: 대부분의 항들이 0인 배열
  - 표현 방법
  <img width="586" height="238" alt="image" src="https://github.com/user-attachments/assets/8eca0d8a-d700-4b11-86e3-f4bc8d586a2a" />
  1. 2차원 배열을 이용하여 배열의 전체 요소를 저장하는 방법.
     - 장점: 행렬의 연산들을 간단하게 구현 가능.
     - 단점: 대부분의 항들이 0인 희소행렬인 경우 많은 메모리 공간 낭비.
       <img width="552" height="167" alt="image" src="https://github.com/user-attachments/assets/d58f4a08-1831-4bc5-908a-5179e6737c4f" />
     - 전치(transpose)행렬 계산하기
       - 2차원 배열의 요소 (i,  j)를 (j, i)와 교환

  2. 0이 아닌 요소들만 저장하는 방법.
     - 장점: 희소행렬의 경우, 메모리 공간의 절약.
     - 단점: 각종 행렬 연산들의 구현이 복잡해진다.
       <img width="543" height="238" alt="image" src="https://github.com/user-attachments/assets/dcb70cfe-8d49-4d1c-a504-bac5c91e0d13" />
     - 전치(transpose)행렬 계산하기
       - (0, 3, 7) 요소 -> (3, 0, 7) 요소
       - (1, 0, 9) 요소 -> (0, 1, 9) 요소
       - 기존 행렬의 열이 행으로 변경
       - 0열에 있는 요소를 모두 찾아서 전치 행렬의 0행으로 저장
       - 1열의 요소를 모두 찾아서 1행으로 저장

3.5 포인터
  - 포인터: 다른 변수의 주소를 가지고 있는 변수
    - 모든 변수는 메모리 주소를 가짐
    - 변수의 주소가 포인터에 저장됨
    - 사용법
      - a: char형 변수
      - p: char형 변수를 가리키는 포인터
      - p가 a를 가리키게 하기 위해 a의 주소를 p에 대입
      - 포인터 p가 가리키는 내용의 변경
  
  - 연산자
    - &(주소 연산자): 변수의 주소를 추출
    - *(간접참조 연산자(역참조 연산자)): 포인터가 가리키는 곳의 내용을 추출

  - 배열의 이름: 배열의 시작위치를 가리키는 포인터
    - 배열의 이름이 있는 곳이 배열의 첫 번째 요소의 주소
    - 배열이 함수의 매개변수로 전달되면 포인터가 전달되는 것

3.6 동적 메모리 할당
  - 프로그램의 실행 도중에 메모리 할당 받는 것
  - 메모리를 효율적으로 사용
    - 필요한 만큼의 메모리를 할당을 받아 사용하고, 사용이 끝나면 시스템에 메모리를 반납  
  - 히프(heap): 동적 메모리가 할당되는 공간
  
  - 동적 메모리 할당 코드
    - malloc() 함수 [엠 얼록]
      - 내가 필요한 만큼 블록 할당
      - 시스템 메모리 부족하면 NULL 반환(NULL 검사 필수)
    - 동적 메모리는 포인터로만 사용 가능
    - free()함수 (필요)
      - 할당된 메모리 블록 변환
     
  - 구조체에 대한 포인터를 선언하고 포인터를 통하여 구조체 멤버에 접근 가능
    - 구조체를 함수로 전달하는 경우 비효율적
    - 구조체 포인터를 전달하는 것이 효율적
   
  - 포인터를 통해 구조체 멤버에 접근하는 표기법
    - (*ps).i
    - ps->i (가독성은 더 좋음)
