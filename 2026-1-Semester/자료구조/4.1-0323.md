4.1 스택이란?
  - 스택(stack): 쌓아놓은 더미
    - 입출력은 맨 위에서만 일어남
    - 스택 중간에서는 데이터를 삭제할 수 없음
   
  - 후입 선출 (스택 제외 대부분은 선입 선출)
    - 가장 최근에 들어온 데이터가 가장 먼저 나감

  - Ex) 시스템 스택을 이용한 함수 호출
    - 함수가 실행이 끝나면 자신을 호출한 함수로 되돌아 감
    - 스택을 사용해서 복귀할 주소를 기억함
    - 함수 호출시 시스템 스택에 활성 레코드 생성
    - 활성 레코드에 복귀 주소, 매개변수, 함수에서 선언되는 지역 변수 생성

  - 스택 구현 방법
    - 배열: 간단, 성능 우수, 스택 크기 고정(단점이 될 수도 있다)
    - 연결 리스트: (배열보다 상대적으로)구현 복잡, 스택 크기 가변적
    - 1차원 배열을 이용한 구현
      - **스택 공백상태: top은 -1**
      <img width="587" height="342" alt="image" src="https://github.com/user-attachments/assets/c9aab847-5583-4d10-ba33-4bcbbb66b068" />

    - 방법1. 전역 변수로 구현
      - 1차원 배열과 top변수를 모두 전역 변수로 구현
      - 참고) return (top == -1) // top이 -1일 때만 return, -1 아니면 false
        
    - 방법2. 구조체 배열로 구현
      - 스택에 저장할 값이 정수나 문자가 아닌 복잡한 구조를 갖는 요소
    
    - 방법3. 구조체 포인터를 함수의 매개변수로 전달
      - stack 배열과 top을 구조체로 결합시키고 구조체 포인터를 함수로 전달
        - Stack Type 구조체를 만들고 stack 배열과 top을 넣는다
        - Stack Type 구조체에 대한 포인터를 각 함수의 매개변수로 전달
        - 각 함수에서 구조체를 포인터로 이용해서 스택을 조작
        - 여러개의 스택을 만들 수 있음

      - 방법1(전역 변수), 방법2(구조체 배열) 장단점
        - 장점: 이해하기 쉽게 구현 가능
        - 단점: 하나의 프로그램에 여러 개의 스택을 동시에 사용하기 어려움
          - stack 배열과 top 변수가 전역변수로 선언

    - 방법4. 스택을 동적 메모리 할당으로 생성
      - 특징
        - 필요시 스택 생성
        - 사용이 끝나면 동적 메모리 반환해야 함

      - 동적 배열 스택
        - 특징
          - malloc()을 호출하여서 실행 시간에 메모리를 할당 받아서 스택을 생성
          - 필요할 때 스택의 크기를 동적으로 늘릴 수 있음

    - 스택의 응용: 괄호 검사
      - 괄호는 쌍이 되게 사용되어야 함

      - 괄호 검사 조건
        - 조건1: 왼쪽 괄호의 개수 = 오른쪽 괄호의 개수
        - 조건2: 같은 관호에서 왼쪽 괄호는 오른쪽 괄호보다 먼저 나와야 한다
        - 조건3: 서로 다른 종류의 왼쪽 괄호와 오른쪽 괄호 쌍은 서로를 교차하면 안된다

      - 알고리즘
        - 스택을 사용하여 왼쪽 괄호를 만나면 삽입하다가 오른쪽 괄호가 나오면 스택에서 가장 최근의 왼쪽 괄호를 꺼내어 타입을 맞춰 본다

        - 절차
          - 문자열에 있는 괄호를 차례대로 조사
          - 마지막 괄호까지를 조사한 후에도 스택에 괄호가 남아 있으면 조건 1에 위배되므로 0(거짓)을 반환하고, 그렇지 않으면 1(참)을 반환한다

    - 스택의 응용: 후위 표기 수식의 계산
      <img width="533" height="91" alt="image" src="https://github.com/user-attachments/assets/5e90f969-ce01-4cfa-bed3-7f479d7df2d1" />

      - 수식 표기방법: 전위(prefix), 중위(infix), 후위(postfix)
      - 컴파일러: 수식 계산을 위해 스택 사용
        - 사람: 중위 표기법 사용
        - 컴파일러: 후위 표기법 사용

      - 컴파일러가 후위 표기식을 사용하는 이유
        - 괄호 필요 없음: (1 + 2) * 7 -> 12+7*
        - 연산자 우선순위 고려하지 않아도 됨: 식 자체에 우선순위 표현됨
        - 수식을 읽으면서 바로 계산
       
      - 후위 표기식 계산(이해가 되어야 함)
        - 수식을 왼쪽에서 오른쪽으로 스캔하여 피연산자이면, 스택에 저장하고 연산자이면 필요한 수만큼의 피연산자를 스택에서 꺼내 연산 실행 및 연산의 결과를 다시 스택에 저장
