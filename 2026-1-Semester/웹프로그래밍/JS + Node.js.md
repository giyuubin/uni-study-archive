  - JS Syntax
    1. 변수 선언 방식
      - 변수는 데이터를 담는 그릇이다. 이들은 각기 작동하는 유효 범위가 다르다
      - var(ES5-구식): 함수 스코프를 가진다. 함수 내부에서 선언된 것만 격리하고, if문, for문 같은 일반적인 블록({})안에서 선언된 변수는 블록 밖으로 누수되어 접근 가능해진다
        - Ex: if (true) { var leaked = "밖에서 보임"; } 이 코드가 실행된 후 console.log(leaked);를 호출하면 에러가 나지 않고 값이 출력됨. 이는 **예측 불가능한 버그의 주범이다**
      - let(ES6-신식): **블록 스코프**를 가진다. {}로 묶인 모든 구역 내에서만 존재하며 밖으로 새어나가지 않는다
        - Ex: let count = 0; count = 1; // 값을 재할당 ㄱㄴ
      - const(ES6): let과 동일하게 블록 스코프 가진다. **상수**이므로 선언과 동시에 반드시 값을 할당해야하며, 이후 절대 값을 재할당할 수 없다
      - **원칙**: 모든 변수는 기본적으로 const를 사용한다. 값이 무조건 바뀌어야만 하는 예외적인 상황에서만 let을 사용하며 **var는 절대 사용 안한다**
    2. 원시 데이터 타입
      - JS에서 객체가 아닌 6가지 데이터 형태
      - string: 텍스트 데이터 // "hello", 'world'
      - number: 정수와 소수점을 구분하지 않고 모두 number로 취급 // 42, 3.14, -1
      - boolean: true, false
      - null: 값이 의도적으로 비어 있음을 개발자가 명시할 때 사용
      - undefined: 변수를 선언만 하고 아직 어떠한 값도 할당하지 않았을 때 시스템이 자동으로 부여
      - typeof: 타입 확인 연산자, 변수의 타입을 확인할 떄 사용 // typenull 실행하면 object가 반환. 버그임
    3. 타입 변환과 동등 연산자 // == vs ===
      - JS는 타입이 다른 두 값을 연산하면, 에러를 내지 않고 지맘대로 타입을 변환한다
      - Ex:
        - "5" + 1 => "51"
        - "5" - 1 => 4 // ??
        - true + 1 => 2
      - 동등 연산자의 차이
        - ==(느슨한 비교): 위처럼 타입을 강제로 맞춤. 0 == false는 타입이 변환돼 true가 됨 // 버그의 원인
        - ===(엄격한 비교): 값뿐만 아니라 타입까지 일치해야 true를 반환한다. 0 === false는 false다
        - **원칙**: 코드를 작성할 때는 반드시 ===만 사용해야한다
    4. 템플릿 리터럴
      - 기존에는 + 남발로 가독성 똥, 그래서 바꿈
      - 구조: 문자열을 **백틱(`)**으로 감쌈
      - 변수 주입: 문자열 중간에 변수나 식 넣고 싶으면 **${변수명}**구조를 사용 // `Hello, ${name}!`
      - 다중 줄: \n 없이 그냥 엔터를 쳐도 형태가 유지되므로 유용함
    5. Nullish 병합 연산자(??)
      - 어떤 변수가 비어있을 때 기본값을 안전하게 부여하기 위한 최신 문법
      - 구조: A ?? B => A가 만약 null이거나 undefined라면 B를 반환하고, 그 외에는 무조건 A를 그대로 반환
  - JS Syntax: Function
    1. 함수의 본질과 일급 객체
      - 함수: 특정한 작업을 수행하는 코드의 묶음. 한 번 정의해 두면 이름을 불러 재사용 ㄱㄴ
      - JS에서 함수 = 일급 객체로 취급 = 함수를 숫자나 문자열 같은 일반 데이터처럼 다룰 수 있다
        - 함수를 변수(const)에 저장 ㄱㄴ
        - 함수를 다른 함수의 인자로 전달 ㄱㄴ
        - 함수 내부에서 또 다른 함수를 결과값으로 return ㄱㄴ
    2. 함수를 정의 하는 3가지 아키텍처 패턴
      - 함수 선언문(구식)
        - function 키워드 먼저 씀
        - 호이스팅: 코드가 실행되기 전 시스템이 함수를 먼저 읽어 끌어올리므로, 함수를 정의하기 전 줄에서도 해당 함수를 호출할 수 있는 특징이 있다 // 오
      - 함수 표현식
        - 이름이 없는 익명의 함수를 만든 후, 이를 const 변수에 할당하는 방식이다
          - Ex: const greet = function(name) { ... };
          - 변수에 담겨 있으므로 호이스팅이 발생하지 않아, 반드시 선언된 이후에만 호출할 수 있어 더 안전하다
      - **화살표 함수(ES6)**
        - Nest.js 프로젝트에서 가장 범용적으로 쓰이는 표준 뼈대
        - function 키워드를 아예 없애버리고, 매개변수와 코드 블록 사이에 (=>)를 넣어 직관적으로 연결한다
          - Ex: const greet = (name) => { ... };
    3. Arrow Function 규칙
      - 화살표 함수는 코드를 더욱 압축할 수 있는 규칙들을 제공한다.
      - 암시적 반환: 함수 내부의 동작이 단 한 줄의 표현식으로 끝난다면, {}와 return 키워드를 통째로 생략 ㄱㄴ. 시스템이 알아서 그 한 줄의 결과를 반환한다
        - 전: const double = (n) => { return n * 2; };
        - 후: const double = (n) => n * 2;
      - 매개변수 괄호 생략: 입력받는 매개변수가 딱 1개라면, 매개변수를 감싸는 ()조차 생략 ㄱㄴ
        - Ex: const double = n => n * 2;
      - 매개변수가 없을 때: 단, 전달받는 값이 아예 없다면 ()는 반드시 작성해야한다
    4. 매개면수 기본값
      - 함수 호출시 필요한 매개변수를 넣지 않으면, JS는 에러를 내는 대신 해당 값을 undefined로 처리한다 // 코드에 버그를 유발함
      - ES6부터 기본값 할당 구조 도입
        - 함수 매개변수 자리에 = 기호 써서 기본값 박아둠
        - Ex: async getRecentCVEs(token, limit = 5) { ... }
        - 만약 호출자가 limit 값을 전달하면 그 값을 쓰고, 아무것도 전달하지 않으면 안전하게 기본값인 5를 사용하게 되는 방어적 구조이다
    5. 콜백 함수
      - 다른 함수에 전달되어, 특정 작업이 끝난 후 실행되도록 시스템에 맡겨진 함수를 Callback이라 부름
      - 메인 함수인 doTask(name, onComplete)가 실행되면 내부에 있는 무거운 로직을 먼저 처리한다
      - 로직이 모두 끝나면, 파라미터로 받아두었던 onComplete가 콜백 함수를 호출하며 결과 데이터를 밀어 넣는다
      - 이 패턴을 통해 코드의 제어권을 안전하게 주고받을 수 있다
  - JS Syntax: Object and Arrays
    1. 구조 분해 할당
      - ES6 기준, 구조 분해 할당은 객체의 포장지를 뜯어내어 내부의 속성들을 한 번에 각각의 변수로 선언하는 구조이다
      - 기본 구조: const { title, severity } = report;
        // 이래 쓰면 report 객체 안에 title과 severity 속성이 동일한 이름의 변수로 단번에 추출된다
      - 추가 규칙
        - 추출하면서 변수 이름을 바로 바꿀 수 있다
          - Ex: const { title: reportTitle } = report;
        - 만약 값이 없을 경우를 대비해 기본값을 설정할 수도 있다
          - Ex: { severity = "LOW" }
    2. 스프레드 연산자
      - 스프레드 연산자는 닉값대로 객체나 배열의 괄호를 허물고 내부 요소들을 개별적으로 펼쳐서 뿌려주는 역할을 한다
      - 사용 목적: 두 객체를 하나로 병합하거나, 원본 객체를 복사하면서 특정 속성만 덮어씌울 때 사용한다
      - 구조적 규칙(우측 우선)
        - const report = { ...base, ...extra }; 구조를 썼을 때, 만약 base와 extra 양쪽에 동일한 이름의 key가 존재한다면, 나중에 덮어씌워진 **우측 extra의 값이 최종적으로 반영**된다
      - Nest.js 아키텍처에서의 활용: 서비스 계층에서 클라이언트가 보낸 요청 데이터를 펼치고, 그 위에 로그인한 유저의 ID를 얹어서 데이터베이스에 저장할 하나의 완벽한 객체를 만들 때 이 패턴이 쓰인다
    3. Array의 기본 제어
      - 배열은 순서가 있는 데이터 집합이다. 실무 웹프에서는 단순한 문자열 배열보다는 객체들을 담고 있는 배열 구조가 가장 널리 쓰인다 // 게시글 목록, 유저 목록 등
      - push(): 배열의 끝에 데이터를 추가한다
      - pop(): 끝에서 데이터를 뽑아낸다
      - length: 배열 내부 요소의 총개수를 반환하는 속성
    4. 배열의 제어(콜백 함수 기반)
      - 화살표 함수(콜백)을 배열 메서드에 던져주면, 배열 안의 모든 데이터를 순회하며 아주 강력한 데이터 조작 ㄱㄴ
      - **주의할 점: map과 filter는 원본 배열을 절대 수정하지 않고, 항상 새로운 배열을 생성하여 반환한다**
      - map(변환): 배열의 모든 요소를 순회하며 콜백 함수를 실행하고, 그 변환된 결과값들만 모아서 새로운 배열 만든다
      - filter(필터링): 콜백 함수의 조건식이 true를 반환하는 요소들만 살아남겨서 새로운 배열로 묶어낸다
      - find(탐색): 조건식이 true가 되는 첫 번째 단일 요소(객체)만 찾아내어 반환하고 순회를 즉시 종료한다. 만약 끝까지 찾아도 없으면 undefined를 반환한다 // 특정 ID값을 가진 단일 게시글을 조회할 때
      - forEach(순회): 결과값을 새로운 배열로 반환하지 않고, 단순히 배열을 처음부터 끝까지 돌면서 콘솔 출력 등의 특정 작업을 반복 수행할 때만 쓴다
  - 비동기 프로그래밍
    1. 동기 vs 비동기
      - 동기(Synchronous): 코드가 위에서 아래로 한 줄씩 실행되며, 앞의 작업이 완전히 끝날 때까지 다음 줄의 코드는 멈춰서 기다린다
      - 비동기(Asynchronous): 시간이 오래 걸리는 작업을 시작시켜 두고, 그 결과를 기다리지 않고 즉시 다음 줄의 코드로 넘어가 실행한다. 이후 백그라운드에서 작업이 완료되면, 시스템이 프로그램에게 알려주어 그때 결과를 처리한다 // Node.js 환경의 웹 서버는 이 비동기 실행이 필수적이며, 이를 통해 단일 프로세스로도 수많은 요청을 멈춤 없이 처리할 수 있다
    2. Callback Hell 발생
      - 과거에는 비동기 작업이 끝났을 때 실행할 코드를 함수 형태로 넘겨주는 콜백 함수 방식을 사용했다
      - 이는 여러 비동기 작업을 순차적으로 연결해야 할 때 심각한 구조적 문제가 발생한다
        - 콜백 지옥: 꼬꼬무 형태가 되어, 코드를 읽기도 어렵고, 유지보수나 에러 처리를 거의 불가능하게 만든다 
        - <img width="631" height="281" alt="image" src="https://github.com/user-attachments/assets/f0fc65ad-a353-408d-801e-75bb5a72a5fd" />
    3. 구조적 해결책1: Promises
      - ES6에서 이를 해결하기 위해 Promise 객체가 도입됨
      - Promise: 비동기 작업의 최종 결과를 담아두는 빈 상자
      - 3가지 상태
        - Pending(대기 중): 작업이 아직 실행 중인 상태
        - Fulfilled(이행됨): 작업이 성공적으로 완료되어 결과 데이터가 상자에 담긴 상태
        - Rejected(거부됨): 작업 중 에러가 발생하여 실패한 상태
      - <img width="814" height="104" alt="image" src="https://github.com/user-attachments/assets/04778284-bbfe-4523-9618-8b0585e20eb0" />

      - 구조적 장점: 깊게 파고들던 콜백 대신, .then()(성공시 처리)과 .catch()(실패 시 에러 처리)라는 메서드를 사용해 아래로 깔끔하게 연결할 수 있게 되었다
    4. 구조적 해결책2: **async / await (현대 표준 아키텍처)**
      - Promise조차 .then()을 계속 연결해야 하는 번거로움 존재
      - ES2017에서 async와 await가 도입됨
      - 구조와 동작 원리
        - 비동기 작업을 수행할 함수 이름 앞에 async 키워드를 붙여 선언
        - 함수 내부에서 시간이 걸리는 Promise 작업(Ex: DB 조회) 앞에 awit 키워드를 붙인다
        - await를 만나면, 해당 코드 줄에서 작업이 완료될 떄까지 내부적으로만 일시 정지한다 // 서버 전체가 멈추는게 아니라 다른 요청은 정상적으로 처리된다

    5. 에러 방어 아키텍처: try / catch
      - async / await 구조에서는 비동기 작업이 실패했을 때 프로그램이 뻗는 것을 막기 위해 반드시 try / catch 블록으로 코드를 감싸야 한다
      - try 블록: 실행할 비동기 로직들을 넣는다. 만약 await한 작업 중 하나라도 에러 발생하면, 그 아래 코드는 모두 Skip
      - catch 블록: 에러가 발생하면 즉시 실행 제어권이 이곳으로 넘어와 안전하게 Error를 처리한다
      - finally 블록(선택): 성공하든 에러가 나든 무조건 마지막에 실행되어야 하는 정리로직을 배치한다

        - 문법: 내보낼 때는 export, 불러올 때는 import 키워드를 사용한다
