- JavaScript
  - Languages
    - HTML: 프로그래밍 언어가 아닌 웹 페이지의 뼈대를 잡는 역할
    - CSS: 레이아웃과 스타일 담당
    - JavaScript/TypeScript: 클라이언트(브라우저) 측에서는 상호작용을, 서버(백엔드) 측에서는 비지니스 로직을 처리하는 핵심 프로그래밍 언어
    - Python/Java: Python은 데이터 기반 백엔드에, Java는 기업용 대규모 백엔드 개발
  - 프레임워크 및 라이브러리
    - 백엔드(Server): JS계열에서는 Express, Nest.js를 사용하며, Python은 Django, Flask, Java는 Spring 사용한다
    - 프론트엔드(Browser): JS기반의 React, Nest.js, Angular, Vue가 표준으로 사용됨
  - ES6 vs ES6+
  - JS 생태계와 런타임
    - 런타임: 프로그래밍 언어가 실행되는 환경을 의미함
    - 과거의 JS: 원래 JS는 브라우저 내부에서만 동작했었다. 브라우저 런타임은 UI 조작을 위해 DOM, window, fetch 등의 API를 제공한다
    - Node.js의 등장
      - 2009년 Ryan Dahl이 크롬 브라우저의 핵심 해석기인 V8 엔진을 떼어내어 독립적인 런타임 Node.js를 개발함
      - 이로인해 JS는 브라우저에서 벗어나 서버 측(Backend) 코드를 작성 가능해짐
