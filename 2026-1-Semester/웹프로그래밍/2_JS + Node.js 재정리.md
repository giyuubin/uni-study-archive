- JS + Node.js

  1. JS 환경의 이해(런타임과 Node.js)
     - 프로그래밍 언어가 실행되려면 Runtime이라는 실행 환경이 필요하다
     - 과거 JS(틀): 오직 브라우저 내부에서만 실행 ㄱㄴ
       // 브라우저 런타임은 화면(UI) 조작을 위한 DOM API 등을 제공함
     - Node.js
       - JS를 브라우저 밖인 서버(Backend)에서도 실행할 수 있게 만든 독립적인 런타임
       - 파일 시스템(fs), 네트워크(http) 등 운영체제 제어 API를 제공한다
     - Node.js의 핵심 아키텍처(이벤트 루프)
       - 전통적인 Java/Spring 서버는 요청마다 새로운 스레드(Thread)를 생성하여 자원 소모가 크다
       - Node.js는 단일 스레드(Single thread)와 비동기(Non-blocking) 방식을 사용한다
       - 시간이 걸리는 작업은 백그라운드에 맡기고 즉시 다음 작업을 처리하다

         1. timers: 예약된 시간 작업 처리
         2. pending callbacks: 대기 중인 콜백 처리
         3. idle, prepare

  2. 프로젝트 뼈대 구축(NPM과 모듈 시스템)
     - 서버 시스템은 수많은 코드가 조립된 형태이다
     - NPM과 package.json
       - package.json: 프로젝트의 이름, 사용할 외부 패키지 목록을 명시한 신분증 역할을 한다
       - node_modules 폴더: NPM을 통해 인터넷에서 다운로드한 외부 코드들이 실제로 물리적으로 저장되는 창고
     - 모듈 시스템(코드를 파일 단위로 쪼개는 기술)
     - <img width="871" height="183" alt="image" src="https://github.com/user-attachments/assets/eaece70e-b1c8-4542-9a07-2357a85b469a" />
       - Common JS(틀): require()로 불러오고 module.exports로 내보낸다.
         코드가 실행되는 도중에 파일을 동기적으로 불러온다
       - ES Modules(현대 표준): import로 불러오고 export로 내보낸다.
         코드 실행 전(Parse time)에 미리 구조를 분석하여 연결한다.
         Node.js에서 쓰려면 package.json에 "type": "module"을 명시해야 한다
     - 우리가 사용하는 Nest.js는 ES Modules 문법(import/export)으로 코드를 작성하면,
       시스템이 알아서 CommonJS로 변환하여 실행해주는 아키텍처를 가진다

  3. JS(ES6+) 기초 문법
     - 변수 선언
       - 버그를 유발하는 var는 절대 쓰지 않는다.
       - 대신 블록 스코프({}) 안에서만 안전하게 머무는 let(재할당 ㄱㄴ)과 const(상수, 재할당 ㅂㄱㄴ)를 사용한다
     - 타입 변환의 위험성과 일치 연산자
       - JS는 0 == false처럼 타입이 달라도 강제로 맞춰서 true로 만들어버리는 느슨한 비교(==)의 위험성이 있다
       - 반드시 타입까지 완벽히 일치해야 하는 엄격한 비교(===)를 사용해야 한다
       - 템플릿 리터럴: 텍스트와 변수를 결합할 때 + 기호 대신 백틱(`)과 ${변수명}을 사용하여 직관적인 구조를 만든다
         // `Hello, ${name}`
       - Nullish 병합 연산자(??)
         - 값이 의도적으로 비어있을 때(null 또는 undefined)만 우측의 기본값을 반환한다
         - 0이나 false 같은 유효한 값을 실수로 덮어씌우는 것을 방지하는 안전장치이다

  4. Funtion 아키텍처와 Callback
     - JS에서 함수는 변수에 담거나 인자로 전달할 수 있는 일급 객체다
     - Arrow Function
       - 현대 백엔드 개발의 표준 뼈대
       - function 키워드 빼고 (매개변수) => { 코드 } 형태로 작성한다
       - 만약 내부 코드가 한 줄의 결과값이라면 중괄호 {}와 return을 생략할 수 있다
     - Callback
       - 다른 함수의 인자로 전달되어, 특정 작업이 끝난 후 실행 권한을 넘겨받아 동작하는 함수
       - Node.js의 비동기 처리 완료 후 주로 호출된다

  5. 데이터 묶음 제어(객체와 배열)
     - 클라이언트의 요청 데이터(JSON)나 데이터베이스 결과를 다루기 위한 뼈대
     - 구조 분해 할당
       - 객체의 포장지를 뜯어 속성을 한 번에 변수로 빼내느 템플릿이다
       - const { title, severity } = report;처럼 작성한다
     - 스프레드 연산(...)
       - 객체나 배열의 괄호를 허물고 내용을 펼친다
       - 원본을 복사한 뒤 새로운 속성을 안전하게 덮어씌워 병합할 때 쓰인다
         (const report = { ...base, ...extra };
     - 배열의 고급 순회 메서드: 원본 배열을 훼손하지 않고 콜백 함수를 이용한다
       - map: 각 요소를 변환하여 새 배열 반환
       - filter: 조건이 참인 요소만 남겨 새 배열 반환
       - find: 조건이 참인 첫 번째 단일 요소 반환
       - forEach: 반환값 없이 각 요소에 대해 단순 작업 반복

  6. 비동기 프로그래밍(Asynchronous) 아키텍처
     - 서버가 멈추지 않게 하는 가장 중요한 논리 구조
     - 동기(Sync) vs 비동기(Async): 동기는 앞줄 작업이 끝날 떄까지 서버가 멈춰서 기다리지만, 비동기는 작업을 던져두고 즉시 다음 줄을 실행하여 다중 클라이언트 요청을 소화한다
     - 현대적 해결책(async/await)
       - 비동기 작업은 Promise(미래의 결과 상자)를 반환한다
       - 이를 쉽게 다루기 위해 함수 앞에 async를 붙이고, 시간이 걸리는 작업 앞에 await를 붙여 해당 줄에만 내부적으로 결과를 기다리게 한다
     - 안전망(try / catch): 비동기 작업 중 에러가 발생하여 서버가 뻗는 것을 막기 위해 비동기 로직을 try블록에 넣고,
       실패 시 catch 블록으로 제어권을 넘겨 에러를 처리하는 아키텍처를 강제하다

  7. Class와 객체 지향 프로그래밍: 관련된 데이터(속성)와 로직(메서드)을 묶어내는 설계도
     - constructor(생성자)와 this: 객체(Istance)가 생성될 때 한 번 실행되며,
       this 키워드로 생성된 객체 자기 자신의 초기 데이터를 세팅한다
     - extends(상속)
       - 부모 클래스의 기능을 자식이 물려받는다
       - 이때 자식이 생성자 내에서는 반드시 super()를 호출해 부모의 생성자를 먼저 작동시켜야 한다
     - static(정적 메서드): 객체를 만들지 않고 클래스 자체에서 직접 호출하는 공용 함수이다
