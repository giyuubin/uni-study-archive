- Web Security
  1. The Web is Everywhere
     - 웹은 전 세계 어디서나 접근 가능한 열린 공간이다
  2. What Makes Web Security Unique?
     - 과거의 전통적인 시스템 웹 보안은 방어해야 하는 타겟이 완전히 다르다
       - 전통적 보안: 주로 운영체제(OS), 커널, 메모리, 바이너리 파일 등을 보호하는 데 집중
       - 웹 보안: 운영체제 위에서 실행되며, 브라우저를 통해 접근하는 애플리케이션 자체를 타겟으로 한다 즉, 사용자가 쓰는 브라우저 그 자체가 공격 표면의 일부가 된다
       - 핵심 원리
         - 웹 앱은 기본적으로 프론트엔드(브라우저), 백엔드(서버), 데이터베이스라는 3계층 구조로 이루어져있다
         - 무서운 점은 공격자가 시스템에 물리적으로 접근하거나 악성코드(Malware)를 심을 필요 없이, 단순히 조작된 HTTP 요청(Request) 하나를 보내는 것만으로도 공격이 성립한다는 것이다
  3. The Attack Chain
     - Vulnerability(취약점): 시스템의 설계, 구현 또는 설정에 존재하는 근본적인 결함. 개발자의 논리적 요류, 입력값 검증 누락, 잘못된 기본 설정 등에서 발생함
     - Exploit: 이 취약점을 이용해먹기 위해 만든 구체적인 기술이나 코드를 의미함
     - Attack: 만들어진 exploit을 실제 타겟 서버에 실행하는 행위
     - Impact: 공격이 성공하여 발생한 최종적인 피해 (Ex: 데이터 탈취, 서비스 중단, 계정 탈취 등)
  4. CVE and CVEE
     - 모든 취약점이 똑같이 위험한 것은 아님. 이를 평가, 식별하는 국제 표준 기준이 있음
