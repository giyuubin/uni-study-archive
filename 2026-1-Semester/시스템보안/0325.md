  - Process Creation)
    - Parent process creates child processes
      - Children can create other process (tree structure)
      - **Each process** has a unique process identifier (PID)
     
    - Resource sharing options
      - Parent and child share all resources (다 공유)
      - Child shares subset of parent's resources (일부만 공유)
      - Parent and child share no resources (공유 안함)

  - Process Creation in UNIX
    - fork(): 부모의 메모리를 그대로 '복제'하여 **새로운** PID를 가진 자식을 만든다
    - exec(): 기존 메모리를 완전히 새로운 프로그램으로 '덮어쓰기'한다 (대체해버림)
    - wait(): makes parent wait for child termination

  - Process Termination
    - Process terminates when it finishes execution
      - Calls exit() system call
      - Returns status value to parent via wait()
        
    - Cascading(겹겹이) termination (부모가 죽으면 다 죽는다)
      - Some OS terminate all children when parent terminates
      - All descendants (children, grandchildren) are terminated
