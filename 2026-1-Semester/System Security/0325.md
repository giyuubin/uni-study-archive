  - Process Creation)
    - Parent process creates child processes
      - Children can create other process (tree structure)
      - **Each process** has a unique process identifier (PID)
     
    - Resource sharing options
      - Parent and child share all resources (다 공유)
      - Child shares subset of parent's resources (일부만 공유)
      - Parent and child share no resources (공유 안함)

  - Process Creation in UNIX
    - fork(): 부모의 메모리를 그대로 '복제'하여 **새로운** PID를 가진 자식을 만든다
    - exec(): 기존 메모리를 완전히 새로운 프로그램으로 '덮어쓰기'한다 (대체해버림)
    - wait(): makes parent wait for child termination

  - Process Termination
    - Process terminates when it finishes execution
      - Calls exit() system call
      - Returns status value to parent via wait()
        
    - Cascading(겹겹이) termination (부모가 죽으면 다 죽는다)
      - Some OS terminate all children when parent terminates
      - All descendants (children, grandchildren) are terminated

  - Static Linking vs Dynamic Linking (대단히 중요한 개념)
    - Static Linking
      - 프로그램이 아직 실행이 안됨
      - All library code copied into executble at compile time
      - Resulting executable is self-contained
      - Larger file size, no external dependencies

    - Dynamic Linking
      - 프로그램이 실행 됨
      - Library code not included in executable
      - Required libraries loaoded at runtime
      - Smaller file size, shared libraries among processes

  - Compilation Pipeline: Step by Step
    - <img width="702" height="300" alt="image" src="https://github.com/user-attachments/assets/560d5739-8358-418e-aaa2-2ebbeabae451" />
      1. Preprocessing(전처리): #include나 #define 같은 메크로를 실제 코드로 확장한다
      2. Compilation(컴파일): 사람이 읽을 수 있는 C코드를 어셈블리어로 번역한다
      3. Assembly(어셈블): 어셈블리어를 기계어로 바꾼다. 이것이 오브젝트 파일(Relocatable file)이다
      4. Linking(링킹): 미완성된 파일과 라이브러리들을 하나로 묶어서 최종 실행 파일(Executable)로 만든다

  - Binary Loading Process
    - When you run a Linux binary
    - Step1: 커널이 EFL 파일을 읽어들여 메모리 공간을 확보
    - Step2: (동적 링킹된 파일의 경우) 커널이 프로그램 코드를 바로 실행하는데 아니라, Dynamic Linker(동적 링커)라는 도우미 프로그램을 먼저 메모리에 부른다
    - Step3: 이 동적 링커가 주도권을 쥐고, 프로그램이 필요로 하는 공유 라이브러리들을 메모리에 전부 올린 다음, 흩어진 메모리 주소들을 서로 연결한다
    - Step4: 모든 조각이 메모리에 세팅되면, main 함수로 점프하여 코드가 실행된다
