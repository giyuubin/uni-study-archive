# 🐧 2단원: Linux Basics
* **Shell (쉘)**: 사용자와 OS 커널 사이의 인터페이스 역할을 하며, 리눅스에서는 주로 **Bash**를 사용합니다.
* **파일 타입 결정**: 리눅스에서 파일의 타입은 확장자가 아닌 파일 내부의 **Header (Metadata/File format)**에 의해 결정됩니다. 
    * 예: `0x7F 'E' 'L' 'F'` (ELF 파일 매직 넘버)
* **주요 디렉토리**:
    * `/etc`: 시스템 설정 파일 (Configuration files)
    * `/bin`, `/sbin`: 필수 실행 바이너리 및 관리자용 바이너리
    * `/var`: 시스템 로그 등 가변 데이터

# 👤 3단원: Linux Account Management
* **Authentication (인증) vs Authorization (인가)**
    * **인증**: 사용자의 신원을 확인하는 과정 (예: `/etc/shadow`의 패스워드 해시 확인).
    * **인가**: 신원이 확인된 후, 자원에 대한 접근 권한을 결정하는 과정 (예: 파일 권한, UID/GID 체크).
* **Principle of Least Privilege (최소 권한의 원칙)**: 사용자와 프로세스에 작업을 수행하는 데 필요한 '최소한의 권한'만 부여해야 한다는 핵심 보안 원칙입니다.
* **UID 분류**: **Root** (UID 0), **System** 계정 (UID 1~999, 보통 `nologin` 쉘 사용), **Regular** 계정 (UID 1000 이상).
* **`/etc/shadow`와 Salt**: 패스워드는 해시로 저장되며, 미리 계산된 해시 테이블인 **Rainbow Table 공격**을 방어하기 위해 랜덤한 **Salt** 값을 추가하여 해싱합니다.
* **SUID (Set-UID)**: (★주관식/단답형 단골)
    * 프로세스 실행 시 권한 검사의 기준이 되는 **EUID(유효 사용자 ID)**가 프로세스 실행자가 아닌 **파일 소유자의 UID**로 변경되는 특수 권한입니다.
    * 예: 일반 사용자가 자신의 비밀번호를 변경할 때 쓰이는 `passwd` 명령어.

# 🛡️ 4단원: Access Control (접근 제어)
* **DAC (Discretionary Access Control, 임의적 접근 제어)**
    * **객체 소유자(Owner)**가 접근 권한을 결정합니다. 유연하지만 악성 프로그램이 사용자의 권한을 그대로 상속받는 **Trojan Horse (트로이 목마)** 공격에 취약합니다.
* **MAC (Mandatory Access Control, 강제적 접근 제어)**
    * **시스템(OS/관리자)**이 보안 라벨(Clearance, Classification)을 비교해 권한을 강제합니다. (예: **SELinux**, **AppArmor**)
* **MLS (Multi-Level Security)**
    * **Bell-LaPadula 모델**: **기밀성(Confidentiality)** 보호가 목적. 
        * **No Read Up** (상위 보안 등급 읽기 금지) / **No Write Down** (하위 보안 등급으로 쓰기 금지). 모토: "Write up, Read down"
    * **Biba 모델**: **무결성(Integrity)** 보호가 목적. (Bell-LaPadula와 반대)
        * **No Read Down** (신뢰할 수 없는 하위 등급 읽기 금지) / **No Write Up** (상위 등급 파일 훼손 금지). 모토: "Read up, Write down"
* **RBAC (Role-Based Access Control)**: 사용자 개인이 아닌 **역할(Role)**에 권한을 부여하여 관리를 용이하게 합니다.

# ⚙️ 5단원: Linux Process
* **Process Memory Layout (메모리 구조)**
    * **Text**: 실행 가능한 기계어 코드 (Read-Only).
    * **Data**: 초기화된 전역/정적 변수.
    * **BSS**: 초기화되지 않은 전역/정적 변수.
    * **Heap**: 동적 할당 메모리 (`malloc`), 낮은 주소에서 높은 주소(위)로 성장.
    * **Stack**: 지역 변수, 함수 인자, 반환 주소. 높은 주소에서 낮은 주소(아래)로 성장.
* **PCB (Process Control Block)**: 프로세스의 상태, 레지스터 값, PC 등을 저장하는 자료구조. CPU가 다른 프로세스로 넘어가는 **Context Switch(문맥 교환)** 시 PCB를 갱신/복원합니다.
* **fork() vs exec()**:
    * `fork()`: 부모의 메모리를 그대로 **복제(Duplicate)**하여 새로운 PID를 가진 자식 프로세스를 생성.
    * `exec()`: 현재 프로세스의 메모리 공간을 아예 새로운 프로그램으로 **교체(Replace)** (PID 유지).
* **동적 링킹 (Dynamic Linking)**: 라이브러리를 실행 파일에 포함하지 않고, 런타임에 메모리에 로드하여 **공유(Share)**하므로 파일 크기와 메모리 사용량을 대폭 줄입니다. (`libc.so`)
* **메모리 단편화**: 
    * **External Fragmentation(외부 단편화)**: 남은 총 메모리는 충분하지만 조각나서 할당할 수 없는 현상. 이를 해결하기 위해 논리적 메모리를 나누는 **Paging(페이징)** 기법을 도입합니다.
    * **Internal Fragmentation(내부 단편화)**: 페이징 사용 시 페이지 프레임 내부에서 남는 자투리 공간이 낭비되는 현상.

# 🔍 6단원: Reverse Engineering (Part 1~3)
* **ELF (Executable and Linkable Format)**: 리눅스의 표준 실행 파일, 공유 라이브러리, 목적 파일 포맷입니다. 파일 오프셋과 메모리에 로드되는 **Virtual Address(가상 주소)**를 구분하는 것이 중요합니다.
* **PLT & GOT (동적 링킹의 핵심)**: (★매우 중요)
    * 외부 함수(예: `printf`)를 호출할 때 런타임에 주소를 찾아 연결하는 **Lazy Binding** 기법에 사용됩니다.
    * **PLT (Procedure Linkage Table)**: 외부 함수 호출을 위한 스텁(Stub) 코드.
    * **GOT (Global Offset Table)**: 런타임에 동적 링커가 찾아낸 실제 함수의 주소가 저장되는 테이블.
* **x86-64 아키텍처 특성**: 가변 길이와 복잡한 명령어를 가지는 **CISC** 구조입니다.
* **System V AMD64 ABI (호출 규약)**: 함수 호출 시 인자를 전달하는 레지스터 순서.
    * **RDI -> RSI -> RDX -> RCX -> R8 -> R9** (7번째부터는 스택 사용)
* **핵심 어셈블리 명령어 & 관용구**:
    * `lea`: 메모리 참조 없이 주소값 자체만 계산. (`mov`는 메모리를 참조함)
    * `test rax, rax`: `rax`가 0인지 판별하는 관용구 (`cmp rax, 0`보다 빠름).
    * `xor rax, rax`: `rax` 레지스터를 0으로 초기화하는 관용구.
    * `call`: 스택에 **Return Address를 Push** 하고 타겟 함수로 점프.
    * `ret`: 스택에서 **Return Address를 Pop** 하여 원래 위치로 돌아감 (`pop rip`와 유사).
* **스택 프레임 (Stack Frame) 구조**:
    * **Prologue (프롤로그)**: `push rbp` (이전 RBP 저장) -> `mov rbp, rsp` (새 RBP 설정) -> `sub rsp, 크기` (지역 변수 공간 할당).
    * **Epilogue (에필로그)**: `leave` (또는 `mov rsp, rbp; pop rbp`로 공간 정리 및 RBP 복구) -> `ret`.
    * RBP를 기준으로 **지역 변수는 음수 오프셋**(`[rbp-0x8]`), **Return Address는 항상 `[rbp+0x8]`**에 위치합니다. (★공격 시 덮어써야 할 핵심 타겟)
* **GDB 디버깅**: `stepi` (함수 내부로 진입) vs `nexti` (함수를 건너뛰고 다음 줄 실행). `x`(메모리 검사).
